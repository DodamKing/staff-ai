# 05. 로드맵

단계별로 쪼개서 진행. 각 버전은 **그 자체로 동작하는 결과물**이어야 함.

## v0 — MVP (최소 동작 버전)

**목표**: YouTube URL → 가사 통합 피아노 반주 악보. 한 곡이라도 끝까지 동작.

### 포함

- [ ] uv 프로젝트 초기화, 기본 의존성 설치
- [ ] `src/staff_ai/` 패키지 골격 + `__init__.py`에 `generate_score()` 노출
- [ ] `audio.py`: yt-dlp로 YouTube URL → wav + 메타데이터(제목, 아티스트)
- [ ] `analysis.py`: librosa로 BPM/beat, autochord(또는 대체)로 코드 진행 추출
- [ ] `analysis.py`: `choose_pattern()` — BPM 임계값 기반 자동 패턴 결정
- [ ] `lyrics.py`: LRClib에서 가사 가져오기 (없으면 None 반환)
- [ ] `patterns/block_chord.py`: 블록코드 반주 패턴 1개 (v0 기본)
- [ ] `score.py`: music21로 MusicXML 생성, 가사 lyric 태그 매핑(LRC 타임스탬프 → 마디)
- [ ] `score.py`: `outputs/{title}.musicxml`로 저장
- [ ] `web/index.html`: 드래그&드롭 영역 + OSMD 렌더
- [ ] 캐시 (오디오/분석 결과)

### 제외 (의도적)

- 사용자 반주 스타일 선택 — 곡에 맞춰 자동 결정
- 가사 음절-음표 정밀 매핑 (라인 단위만)
- 키 변경
- 음원 재생/동기화
- 사용자 편집
- HTTP 서버, CLI 도구 (전혀 만들지 않음)

### 완료 기준

좋아하는 한국 발라드 한 곡으로 Claude Code에 URL 던지기 → 1분 이내에 `outputs/`에 MusicXML 저장 → 브라우저 뷰어에 드래그&드롭 → 악보 + 가사 표시. 코드 진행이 70%+ 정확하고 가사가 대략 맞는 마디에 위치.

## v1 — 품질 / 자동 매칭 개선

v0가 동작한다는 전제하에. **사용자 선택지를 늘리는 게 아니라, 곡에 맞는 패턴을 더 잘 고르는 방향.**

- [ ] 패턴 추가: `arpeggio` (느린 곡), `eight_beat` (빠른 곡)
- [ ] `choose_pattern()` 개선 — BPM 외에 코드 변화 빈도, 메이저/마이너 키 반영
- [ ] 코드 추정 정확도 개선 — 후처리(코드 변경 빈도 제약, 곡의 키 기반 후보 좁히기)
- [ ] 곡 구조 인식 (verse / chorus) — 구조 경계에서 패턴 살짝 변주
- [ ] 가사가 LRClib에 없을 때 fallback (제목/아티스트 수동 지정해서 재검색, 또는 가사 없이 진행)
- [ ] 더 나은 코드 detector 비교/교체 (autochord → chord-extractor 등)

## v2 — 사용성 / 정밀도

- [ ] 키 변경 (음역대 조정) — music21 transpose 한 줄
- [ ] 가사 음절-음표 정밀 매핑 (멜로디 채보 없이 박자 분할로 근사)
- [ ] PDF 다운로드 (music21 → LilyPond/MuseScore 경유)
- [ ] 진행률 표시 (에이전트가 단계별로 알려주기)
- [ ] 결과 검토/재생성 루프 (에이전트와 대화로 "이 부분 코드 다시" 같은 요청)

## v3 — 확장 가능성 (할지 안 할지 미정)

- [ ] 다른 악기 (기타 코드 다이어그램 동시 표시)
- [ ] 다국어 (영어 팝송 등)
- [ ] 모바일 친화 뷰어 UI
- [ ] 정적 뷰어를 GitHub Pages 등에 호스팅 (개인용 링크 공유)

## 우선순위 원칙

1. **한 곡이라도 끝까지** 동작하는 게 우선. 모든 모듈을 80%로 만들기보다 한 흐름을 95%로.
2. v0에서 막히면 **스코프 줄이기** 우선 (예: 가사 없이 코드만, 또는 패턴 하나만)
3. **사용자 선택지를 늘리지 않는다.** 자동 결정 정확도를 높이는 방향.
4. 라이브러리 교체보다 **로직 개선** 우선

## 비목표 (계속)

이 프로젝트가 절대 가지 않을 방향:

- 상용화 / 멀티 유저 / 클라우드 배포
- 풀 채보 (멜로디 + 모든 악기)
- 실시간 / 라이브 연주
- 모바일 앱
- HTTP 백엔드 서버
- CLI 도구
- 반주 스타일 사용자 선택 (장르를 바꾸는 식의 옵션)
