# 01. 개요

## 한 줄 요약

YouTube URL을 주면 **그 노래의 피아노 반주 악보**(가사 포함)를 자동으로 만들어주는 도구. Claude Code 같은 에이전트에게 자연어로 요청하면 로컬에서 분석/생성하고, 결과 MusicXML을 정적 웹 뷰어에서 표시.

## 문제

대중가요를 부르고 싶을 때, 피아노로 반주해줄 악보가 필요해.

- 멜론악보, 마장뮤직 같은 유료 사이트는 곡당 결제 + 모든 곡이 다 있는 것도 아님
- 무료 사이트는 코드만 있고 (기타용) 피아노 반주 패턴은 없음
- 직접 채보? 오래 걸리고 음악이론 지식 필요

## 해결 방식

**전체 채보 대신 "코드 + 패턴" 접근.**

대중가요는 90%가 4코드 진행 + 단순한 구조. 그래서:

1. YouTube 오디오에서 **코드 진행, BPM, 박자만** 추출 (어려운 부분 우회)
2. **곡 분위기에 맞는 피아노 반주 패턴**(아르페지오/블록코드 등)을 알고리즘으로 자동 결정 + 적용
3. **가사**는 LRClib에서 가져와서 악보에 통합 (`<lyric>` 태그)
4. MusicXML 파일로 저장 → 정적 웹 뷰어에서 표시

→ 풀 채보(보컬 멜로디까지 정확히 옮기는 것)는 시도하지 않음. 사용자가 직접 노래 부르므로 멜로디 라인은 악보에 불필요.

→ 반주 스타일은 **사용자가 고르지 않음.** 원곡의 BPM/분위기에 맞춰 자동 결정. ("발라드를 재즈풍으로" 같은 옵션은 본 도구 용도와 안 맞음)

## 사용자

- **본인 (개인용)**. 상용 아님.
- Claude Code (또는 동등한 코딩 에이전트) 사용 환경.
- 노래 부르고 싶고, 옆에서 피아노 칠 사람이 있거나 직접 치는 상황.

## 사용 시나리오

1. 부르고 싶은 노래의 YouTube 링크 복사
2. Claude Code 열고 "이 URL 악보 만들어줘: https://youtu.be/..." 라고 자연어로 요청
3. Claude가 staff-ai 모듈을 호출해서 다운로드 → 분석 → MusicXML 생성 (`outputs/`에 저장)
4. 웹 뷰어(`web/index.html`)를 브라우저로 열고 생성된 파일 드래그&드롭 → 악보 + 가사 표시

## 왜 에이전트 + 정적 뷰어 구조인가

- **YouTube 추출은 로컬에서만.** yt-dlp는 클라우드 서버 IP에서 자주 차단당함. 로컬 실행이 가장 안정적.
- **웹은 정적 파일만.** 백엔드 서버가 없으니 GitHub Pages 같은 데 올려도 되고, 그냥 로컬 파일로 열어도 됨. 서버 운영 비용/번거로움 0.
- **사용자 인터페이스는 자연어.** CLI 플래그 외울 필요 없고, Claude가 "이 곡 가사가 LRClib에 없네요. 어떻게 할까요?" 같은 상황도 자연어로 대응 가능.

## 비목표 (지금 안 하는 것)

- 보컬 멜로디 채보 (사용자가 부를 거라 불필요)
- 음원 재생/동기화 (단순 악보 표시 도구)
- 반주 스타일 사용자 선택 (자동 결정만)
- 키 변경 (v0 기준 — 미래에 추가 검토)
- 다른 악기 (기타, 드럼 등)
- HTTP 백엔드 서버 (FastAPI 등)
- CLI 도구 (에이전트가 함수 호출하면 됨)
- 모바일 앱 / 상용 배포
- 사용자 편집 기능 (생성된 악보 수정)
