# 02. 배경

## 왜 만드는가

부르고 싶은 노래의 피아노 반주 악보를 손쉽게 얻고 싶음. 기존 방법은 다 한계가 있음.

## 기존 솔루션의 한계

### 유료 악보 사이트 (멜론악보, 마장뮤직, 음악서비스 등)

- 곡당 결제 ($2~5/곡)
- 인기곡 위주, 마이너한 곡은 없음
- 편곡 스타일이 고정 (내가 원하는 패턴 아닐 수 있음)

### 무료 코드 사이트 (코드모아, 가호악보, Ultimate Guitar 등)

- 코드 + 가사만 있음 (기타용)
- 피아노 반주 패턴은 없음 → "C 코드"라고만 적혀있고 어떻게 칠지는 사용자가 알아서

### 자동 채보 도구 (Klangio, RipX, AnthemScore 등)

- 풀 채보 시도 → 대중가요 풀밴드 사운드는 결과 품질 들쭉날쭉
- 멜로디는 그럭저럭, 반주는 엉망 (드럼/베이스/기타가 다 섞임)
- 유료, 대부분 데스크톱 앱

### 직접 채보

- 음악이론 + 청음 능력 필요
- 한 곡당 몇 시간~며칠

## 우리의 차별점

**"풀 채보 안 함. 코드 + 패턴으로 갈음."**

- 코드 진행 추출은 풀밴드 사운드에서도 잘 됨 (단순 chroma 기반 알고리즘으로도 80%+ 정확도)
- 반주 패턴은 알고리즘으로 깔끔하게 생성 → 일관된 품질 보장
- 단점: 원곡의 디테일(특정 리프, 솔로 등)은 반영 안 됨 → 하지만 노래 부를 용도엔 충분

## 왜 지금 가능한가

몇 년 전이면 어려웠을 부분들이 지금은 가능:

| 작업 | 현재 가능하게 한 것 |
|---|---|
| YouTube 오디오 추출 | yt-dlp (활발히 유지보수됨) |
| 코드 진행 추출 | librosa chroma + 여러 오픈소스 chord detector |
| BPM/박자 추출 | librosa beat tracking |
| 가사 + 타이밍 | LRClib (무료 오픈 API) |
| 악보 데이터 처리 | music21 (MIT, 성숙한 라이브러리) |
| 웹에서 악보 렌더링 | OpenSheetMusicDisplay (MusicXML 표준 지원) |

→ 모든 조각이 무료/오픈소스로 존재. 우리가 할 일은 **이어 붙이고 반주 패턴 생성 로직 만드는 것**.

## 저작권 관련

- **개인 사용 한정.** 상용/배포 없음.
- YouTube 다운로드는 ToS상 회색지대지만, 개인 학습/연주 목적의 도구는 흔히 존재.
- LRClib 가사는 커뮤니티 라이선스. 개인 사용에 문제 없음.
- 만약 나중에 공개/배포로 방향이 바뀌면 이 결정사항 전부 재검토 필요.
